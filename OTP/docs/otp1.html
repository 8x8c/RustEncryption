OTP File Encryption/Decryption Utility

Overview

This command-line utility, written in Rust, performs XOR encryption or decryption on an input file using a key file named key.key located in the current directory. The tool reads the entire input file into memory, processes it with the XOR operation using the key, and outputs the result.

Features

  • Simple XOR encryption/decryption of files.
  • Two operation modes:
    • Normal Mode: Specify both input and output file paths.
    • Atomic Overwrite Mode: Use the -over flag to safely overwrite the input file.
  • Atomic file replacement in overwrite mode ensures data safety by writing to a temporary file first before replacing the original.

Usage

Normal Mode

In normal mode, provide the input file and output file as command line arguments: 

otp <input_file> <output_file>

The tool reads the input_file, applies the XOR operation, and writes the output to output_file.

Atomic Overwrite Mode

To overwrite the input file atomically, use the -over flag followed by the input file: 

otp -over <input_file>

In this mode, the program writes the processed data to a temporary file (in the same directory) and then atomically renames it to replace the original input file.

Compilation

To compile the program, ensure you have Rust installed. Then use Cargo to build the release version: 

cargo build --release

The executable will be located in the target/release directory.

Notes

  • The key file (key.key) must be present in the current directory.
  • The key file must be at least as large as the input file; otherwise, the program will exit with an error.
  • This program reads entire files into memory. Ensure you have sufficient memory for large files.
